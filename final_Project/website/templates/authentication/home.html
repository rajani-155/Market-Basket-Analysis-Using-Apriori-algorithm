What is Market Basket Analysis?

In today's competitive marketplace, understanding customer behavior and preferences is crucial for businesses striving to succeed.

Market Basket Analysis is a statistical data mining technique used by retailers to increase sales by better understanding customer purchasing patterns.

By hunting through transactional data and uncovering hidden relationships between products, businesses can unlock valuable insights about their customers’ purchasing patterns and make informed decisions to optimize marketing strategies, improve cross-selling efforts, and enhance customer satisfaction.

Apriori Algorithm

Apriori algorithm is a classic algorithm used for frequent itemset mining and association rule learning in data mining and machine learning. It is primarily used for finding associations or relationships between items in a dataset. The algorithm is named after the Latin phrase "a priori," meaning "from the former" or "from the previous."

The goal of the Apriori algorithm is to discover frequent itemset in a given dataset, where an itemset is a collection of items that frequently occur together. It follows the principle of the "apriori property," which states that if an itemset is frequent, then all of its subsets must also be frequent.

Support

The support of an item is the number of transactions containing the item. Those items that do not meet the minimum support are excluded from the further processing.

Support (A) = Total Number of Occurrence (A)/Total Number of Transactions

Where, N represents the total number of baskets/transactions(N)

Confidence

Confidence is defined as the conditional probability that a transaction containing the LHS will also contain the RHS.

Confidence (A → B) = Support Count (A U B)/Support Count (A)

Lift

Lift refers to the increase in the ratio of sale of item B when item A is sold.

Lift (A → B) = Support Count (A U B)/ Support Count (A) * Support Count(B)

Graphs

Bar Plot

A Bar plot, also called Bar Chart uses either horizontal or vertical bars (column chart) to show discrete, numerical comparisons across categories. One axis of the chart shows the specific categories being compared and the other axis represents a discrete value scale.

The length of each bar is proportional to the value of the variable at that point in time.

The chart plots the value vertically whereas we perceive the value to be at right angles to the general direction of the chart.

Scatter Plot

A Scatterplot places points on a Cartesian Coordinates system to display all the values between two variables. It is used for showing the relationship between two different variables i.e., correlation and distribution analysis.

The strength of the correlation can be determined by how closely packed the points are to each other on the graph. It helps in spotting anomalies or outliers.

When similarities in the data set are to be highlighted, and also when we have too much data to quickly see a pattern then, it is best. It is ideal when we have paired numerical data and want to see if one variable impacts the other.

Parallel Plot

Each variable is given an axis and all the axes are placed parallel to each other and values are plotted as a series of lines that are connected across all the axes. Each line is a collection of points placed on each axis, that have all been connected.

The order the axes are arranged can impact the way how the reader understands the data. The relationships between adjacent variables are easier to perceive than for non-adjacent variables. Re-ordering the axes can help in discovering patterns or correlations across variables.

It is used for plotting multivariate, numerical data and comparing many variables together and seeing the relationships between them.

Heatmap

Heatmap depicts a pair-wise correlation matrix obtained from different data attributes. Typically, all the rows are one category and all the columns are another. The cells are the intersections of the rows and columns, which can either contain categorical data or numerical data.

A legend is placed alongside a Heatmap for it to be successfully read. Categorical data is color-coded, while numerical data requires a color scale that blends from one color to another, in order to represent the difference in high and low values. It provides us with an easy to understand visual representation of high, negative or none correlation, shows variance across multiple variables and reveals any patterns.

Network Diagram

Also known as Network Graph, Network Map, Node-Link Diagram. It shows how things are interconnected through the use of nodes and link lines to represent their connections and help illuminate the type of relationships between a group of entities.

Typically, nodes are drawn as little dots or circles, but icons can also be used. Links are usually displayed as simple lines connected between the nodes. However, in some Network Diagrams, not all of the nodes and links are created equally by making the node size or link stroke weight proportion to an assigned value.

Network Diagrams can be used to interpret the structure of a network by looking for any clustering of the nodes, how densely nodes are connected or how the diagram layout is arranged.The two notable types of Network Diagrams are "undirected" and "directed". Undirected Network Diagrams only display the connections between entities, while directed Network Diagrams show if the connections are one-way or two-way through small arrows.
